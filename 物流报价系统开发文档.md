请帮我开发一个「物流报价智能查询系统」。该系统需要读取特定结构的 Excel 报价单并将其转换为结构化数据（或轻量级数据库），然后允许用户通过前端界面（或命令行）进行智能组合查询。

以下是该系统的详细需求、数据结构以及业务逻辑：

### 一、 系统核心功能需求
1. **数据导入**：系统能够上传/读取每周更新的 Excel 报价单文件。由于每周表结构可能微调，定位数据时请基于“表头关键字匹配”而不是写死绝对行列号。
2. **多仓库组合查询**：用户可输入 1 至 5 个 Amazon 仓库代码（至少输入一个，如：ONT8, LAX9, LGB8），系统需同时检索出这几个仓库在所有有效渠道中的报价与时效。
3. **时效过滤（可选输入）**：用户可以输入一个数字（代表期望的时效天数，例如 25，只要这个时间在渠道对应的时间段内代表这个渠道可行，如果有多个可行渠道就取最便宜的）。
   - 如果输入了时效数字：输出一组【满足该时效要求】的渠道方案。
   - 如果没有输入时效：则不输出这组过滤方案。
4. **最优化方案推荐**：无论是否输入时效，系统必须针对输入的仓库组合，自动计算并输出以下两组方案：
   - **最优价格方案**：为这 1-5 个仓库匹配出的【总价格最便宜】的渠道组合（单独列出每个仓库用什么渠道最便宜，以及总价）。
   - **极致时效方案**：为这 1-5 个仓库匹配出的【时效最快（天数上限或下限最小）】的渠道组合。

---

### 二、 Excel 数据源及子表解析规则
报价单包含很多子表，但只有以下表格为**有效渠道表**，其他子表（如目录、说明、偏远代码等）请自动忽略。

#### 1. 特殊渠道表：「直送专线」
- **渠道名称判定**：不要使用整个表格的名字“直送专线”作为渠道名。在此表中，仓库代码左侧通常有一列为具体的细分渠道名（如“德宁专线-美西直送”，是合并了多个单元格），请以该仓库代码左侧的名称作为最终的渠道名称。
- **定位规则**：表头“仓库代码”通常在第 3 第 4 行的第 2 列（有合并单元格）。请通过遍历前几行，动态检索包含“仓库代码”字样的单元格，其下方的非空单元格并且使用正则表达式检索四位大写字母即为仓库代码。
- **价格提取逻辑**：
  - 在“仓库代码”右侧，只有“东莞/广州/泉州”的价格列（合并单元格），只需提取“含税价/XXKG+”这一列的公斤价格**，忽略按方（CBM）计费的不含税价格。
  - 价格列通常分为两行细分表头，包含“含税价/21KG+”（或“含税价/10KG+”）和“不含税/1CBM+”。
- **时效提取与数字化逻辑**：
  - 表中有一列名为“参考时效（开船后）”或“参考时效”。
  - 文本格式通常为：`开船后13-17天左右，船开当天...` 或 `船开20-30天左右...`。
  - **请使用正则表达式（Regex）从文本中提取出数字时间段**（例如：将“开船后13-17天左右...”转换为上限 `17` 天，下限 `13` 天），以便后续用于时效过滤和最快方案的排序（排序时优先按时效上限天数升序排列）。

#### 2. 通用渠道表（其余 11 个有效表）
包含以下子表（表名即为渠道名）：
「安达闪线」、「安达迅线」、「电商快线」、「美西盐田特惠」、「德宁专线」、「全美卡派（非热门仓库）」、「奥克兰专线」、「休斯顿专线」、「芝加哥专线」、「美东快船专线」、「德宁专线-萨凡纳」、「德宁专线-美东拆柜」
- **定位规则**：表头“仓库代码”通常在第 3 第 4 行的第 1 列（合并单元格）。请编写逻辑动态查找包含“仓库代码”的列。
- **价格提取逻辑**：
  - 在“仓库代码”右侧，会有对应起运地（如“东莞/广州/泉州”、“义乌”、“上海”）的价格列。
  - 价格列下方通常分为两行细分表头（合并单元格），包含“含税价/21KG+”（或“含税价/10KG+”）和“不含税/1CBM+”。
  - **系统只需提取“东莞/广州/泉州”“含税价/XXKG+”这一列的公斤价格**，忽略按方（CBM）计费的不含税价格。
- **时效提取与数字化逻辑**：
  - 表中有一列名为“参考时效（开船后）”或“参考时效”。
  - 文本格式通常为：`开船后13-17天左右，船开当天...` 或 `船开20-30天左右...`。
  - **请使用正则表达式（Regex）从文本中提取出数字时间段**（例如：将“开船后13-17天左右...”转换为上限 `17` 天，下限 `13` 天），以便后续用于时效过滤和最快方案的排序（排序时优先按时效上限天数升序排列）。

### 三、 算法与输出格式要求
当用户输入 3 个仓库（例如：`ONT8, WM-LAX1, TEB6`）和时效 `30` 时，系统应按如下格式输出：

1. **各仓库全渠道报价一览表**：
   展示每个输入的仓库在所有有效渠道里的价格和时效区间。
2. **时效达标推荐方案（仅在输入数字时显示）**：
   筛选出时效上限 $\le 30$ 天的所有渠道及价格。
3. **🚀 极致省钱方案（5仓/多仓最优价格组合）**：
   - 仓库 A (ONT8) -> 推荐渠道：[德宁专线-美西直送] | 来源表：[直送专线] | 单价：X 元/KG | 时效：25-35天
   - 仓库 B (WM-LAX1) -> 推荐渠道：[XXXX] | 来源表：[XXXX] | 单价：Y 元/KG | 时效：XX天
   - 总计预估单价均值/总和。
4. **⚡ 极致时效方案（5仓/多仓最快时效组合）**：
   - 仓库 A (ONT8) -> 推荐渠道：[安达闪线] | 来源表：[安达闪线] | 单价：M 元/KG | 时效：13-17天
   - ...

### 四、 技术栈建议
- 请使用 Python 开发。
- 使用 `pandas` 和 `openpyxl` 处理 Excel。由于存在合并单元格，读取时请注意填充处理，确保合并单元格拆分后每一行都能正确关联渠道名或表头。
- 可以提供一个简单的 Web 界面（如使用 `Streamlit` 或 `Gradio`），方便我直接拖拽上传 Excel 并输入仓库代码进行查询。
